# [和弦播放器]

## Proposal Report

### 動機與目標
動機:
因為我在家裡彈琴的時候，想要在一組我想要的伴奏上彈點東西，我都必須要先自己錄，然後再循環播放，很麻煩。所以我想要寫出一個能夠靈活調整的和絃播放器。
目標:
現在網路上有不同的算法去模擬樂器的音色，我想要比較不同的算法他們實際產生的聲音各有何不同，還有他們分別的延遲(主要是他們算出samples要花多少時間)。
### 預期功能
<!-- 列出預計實作的功能 -->
我希望它可以依據使用者的輸入來循環播放和弦，使用者可以選擇以幾個和弦為一組，播甚麼和弦，用甚麼音色，要播多久，要用甚麼速度播放等等。在播放的同時，也會顯示吉他紙板上哪些音可以彈那些不能。
### 使用技術
<!-- 使用的語言、框架、工具等 -->
我會用c++來寫這個專題，環境我會用vscode。
我的框架:我主要會設計兩個class，一個是和弦，另一個是樂器(音色)。和弦的成員變數有和弦名子、組成音、那些音的頻率等等。成員函式有播放和getter、setter等等。樂器的話會先有一個instrument當parent，然後在往下完成其他樂器，把各自的聲音合成方式寫出來。然後最後會有一個global function來統合播放。我會引入SDL這個函式庫，來合成波型發出聲音。

### 競品比較
現在編曲軟體要弄出一段簡單的伴奏，都要自己一個音一個音慢慢pick出來，雖然用編曲軟體可以做的比較細緻，就是可以把甚麼時候要有甚麼音都安排好，但我有時候笨笨的要一個一個pick那些音
我會花很久時間，而我這個不同的點是只要打和弦代號他就會自己pick好那些音(雖然我沒有辦法精確要求哪一拍要有哪些音)。
### Prototype 預計可驗證內容
預計可以把輸入和弦代號然後播出和弦這個功能做好，只是音色的模擬還需要調整。

## Final Report

# 專案說明

我這次製作的是一個和弦播放器（Chord Player），使用者可以根據自己的需求，決定播放的和弦進行、樂器種類、速度（BPM）、音色演算法以及效果器參數等。

和弦可以視為多個音符同時發聲的組合，而每個音符都對應到特定的頻率。因此，聲音合成的核心目標就是根據這些頻率產生對應的波形，再將它們混合成最終的聲音。

本程式採用數位音訊的方式產生聲音，設定採樣率（Sample Rate）為 44100 Hz，也就是將一秒鐘切分成 44100 個時間點，並在每個時間點計算聲波的振幅值。算出每個時間點的振幅後，就能形成可以播放的數位音訊 sample。

## 程式流程

我的程式流程如下：

使用者輸入 → 根據輸入得出頻率和各參數資料 → 用樂器算法得出一段基礎音訊 → 疊加效果器算法得出最終音訊 → 將音訊傳進播放函式並播放

本專案依照樂器種類分為 Guitar、Piano 與 Rhodes 三組，每組各實作兩種不同的聲音合成演算法。這些演算法的目的都是根據輸入的頻率資料產生對應的波形，再將多個音符合成為最終聲音。不同演算法的差異在於它們模擬樂器聲音的方式不同，因此在音色特性與運算成本上也有所不同。

# 一、Guitar（吉他）

## 1. Karplus–Strong Synthesis

Karplus–Strong 演算法是一種經典的撥弦樂器合成方法，常被用來模擬吉他的弦振動。它的核心概念是利用延遲線（Delay Line）（琴弦過去一個週期內的振動狀態）與回授（Feedback）來模擬琴弦在被撥動後來回振動並逐漸衰減的行為。

核心公式：

y[n] = 0.5 × (y[n-N] + y[n-N-1])

其中 N 約等於 fs / f，fs 為取樣率，f 為音符頻率。

每次從 buffer（延遲線）取出兩個相鄰 sample 做平均，再放回尾端。由於平均的過程會逐漸消除高頻成分，因此聲音會隨時間慢慢變得柔和並自然衰減，形成類似真實吉他弦的聲音。

時間複雜度：O(SK)

其中：

* S 為總 sample 數量
* K 為同時發聲的音符數量

空間複雜度：O(KL)

其中：

* L ≈ fs / f
* 代表每個音符所需的延遲線長度

Benchmark：

產生一個 C4_M7、tone=2.0、delay=0、bpm=120、bar=1 的音訊所需時間（9 次平均）：

15.866 ms

---

## 2. Extended Karplus–Strong Synthesis

Extended Karplus–Strong 是對基本 Karplus–Strong 的改良版本，目的是讓聲音更接近真實吉他，並改善基本模型的一些限制。

此版本加入：

* Fractional Delay
* Damping
* Pick Position

核心公式：

current = (1 - α)x0 + αx1

next = feedback × ((1 - d)current + d × prev)

其中：

* x0：buffer 目前位置的 sample
* x1：下一格 sample
* α：插值比例
* d：damping 參數
* feedback：回授係數

Fractional Delay 用來修正非整數延遲長度造成的音高誤差。

Damping 用來模擬高頻比低頻更快衰減的現象。

Pick Position 則模擬不同撥弦位置所造成的頻譜差異。

目前 Pick Position 為固定值，未來可擴充為使用者可調參數。

時間複雜度：O(SK)

空間複雜度：O(KL)

Benchmark：

9.594 ms

### 觀察

1. 兩者聽感差異不大。
2. 雖然 Extended Karplus–Strong 理論上運算較複雜，但實測速度反而較快。推測原因是算法 1 使用 deque，而算法 2 使用 vector 搭配 circular buffer，具有較佳的記憶體區域性，因此降低了記憶體存取成本。

# 二、Piano（鋼琴）

## 1. FM Synthesis

FM（Frequency Modulation）Synthesis 是一種利用兩個振盪器合作產生聲音的方法。

* Carrier：產生最終聲音
* Modulator：調變 Carrier

核心公式：

y(t) = sin(ωc t + I sin(ωm t))

其中：

* ωc：Carrier 頻率
* ωm：Modulator 頻率
* I：Modulation Index

FM 能夠透過少量參數產生大量泛音，因此非常適合模擬鋼琴等頻譜較複雜的樂器。

時間複雜度：O(SK)

空間複雜度：O(K)

Benchmark：

40.234 ms

---

## 2. Additive Synthesis with Inharmonicity

Additive Synthesis 是透過產生多個不同頻率的正弦波，再將其相加形成聲音。

核心公式：

y(t) = Σ An sin(2πfn t)

其中 n 表示第 n 個泛音。

為了模擬鋼琴弦剛性所造成的頻率偏移，加入 Inharmonicity：

fn = nf √(1 + Bn²)

其中：

* B 為 Inharmonicity 係數

時間複雜度：O(SKH)

H 為 harmonic 數量（本專案 H=7）。

空間複雜度：O(KH)

Benchmark：

152.995 ms

### 觀察

1. 兩者聽感差異不大。
2. FM 只需計算兩個振盪器，而 Additive 需額外計算 7 個泛音，因此運算成本顯著較高。

# 三、Rhodes（電鋼琴）

## 1. Additive Synthesis

Rhodes 電鋼琴的聲音來自金屬音叉（Tine）振動後，再透過拾音器轉換成電訊號。

本專案透過多個 partial 疊加模擬其音色。

核心公式：

y(t) = Σ An e^(-λn t) sin(2πrnft)

其中：

* An：第 n 個 partial 振幅
* λn：第 n 個 partial 衰減速度
* rn：第 n 個 partial 頻率比例

時間複雜度：O(SKP)

P 為 partial 數量（本專案 P=6）。

空間複雜度：O(KP)

Benchmark：

287.852 ms

---

## 2. Modal Synthesis

Modal Synthesis 屬於物理建模方法，將聲音視為多個振動模態（Mode）的組合。

核心公式：

y(t) = Σ Am e^(-λm t) sin(2πfm t)

其中：

* Am：第 m 個模態振幅
* λm：第 m 個模態衰減速度
* fm：第 m 個模態頻率

時間複雜度：O(SKM)

M 為 mode 數量（本專案 M=6）。

空間複雜度：O(KM)

Benchmark：

195.626 ms

### 觀察

1. 兩者音色差異較大，因此不易直接比較。
2. Rhodes Additive 額外加入 click、attack、airy 與 tremolo 等音色修飾項，因此常數成本較高。

# 四、效果器

## 1. Tone（音色亮度控制）

### 功能

Tone 用來調整聲音亮度（Brightness）。

* Tone 較小：削弱高頻，使音色較暗
* Tone 較大：強化高頻，使音色較亮

### 實作方法

當 Tone < 1 時：

y[n] = a x[n] + (1-a)y[n-1]

當 Tone > 1 時：

y[n] = x[n] + g·hp

其中：

hp = x[n] - x[n-1]

時間複雜度：O(N)

空間複雜度：O(N)

---

## 2. Delay（回聲效果）

### 功能

Delay 用來模擬回聲（Echo）效果。

### 實作方法

y[n] = x[n] + k·x[n-D]

其中：

* x[n]：原始訊號
* x[n-D]：延遲後訊號
* k：衰減係數

時間複雜度：O(N)

空間複雜度：O(N)

---
