# [專題名稱]
和弦播放器
## Proposal Report

### 動機與目標
<!-- 說明為什麼想做這個專題 -->
因為我在家裡彈琴的時候，想要在一組我想要的伴奏上彈點東西，我都必須要先自己錄，然後再循環播放，很麻煩。所以我想要寫出一個能夠靈活調整的和絃播放器。
### 預期功能
<!-- 列出預計實作的功能 -->
我希望它可以依據使用者的輸入來播放和弦，使用者可以選擇以幾個和弦為一組，播甚麼和弦，用甚麼音色，要播多久，要用甚麼速度播放等等。在播放的同時，也會顯示吉他紙板上哪些音可以彈那些度能。
### 使用技術
<!-- 使用的語言、框架、工具等 -->
我會用c++來寫這個專題，環境我會用vscode。
我的框架:我主要會設計兩個class，一個是和弦，另一個是樂器(音色)。和弦的成員變數有和弦名子、組成音、那些音的頻率等等。成員函式有播放和getter、setter等等。instrument會先有一個parent，然後在往下完成其他樂器來把各自的聲音合成方式寫出來。然後最後會有一個global function來統合播放。合成聲音我會用SDL引入這個函式庫。

### 時程規劃
<!-- 各週預計完成的進度 -->
在前4周，完成那兩個class並確認他們的函式功能正常(例如可以正確發出聲音)
後4周完成顯示指板這個功能、統整之前功能然後把輸入輸出搞定，最後拍影片甚麼的。
### 與課程的關聯
<!-- 你的專題可能涉及哪些資料結構或演算法概念？為什麼？ -->
在我的兩個 class 中，會用到課程教的資料結構。Chord class 使用 vector 儲存頻率列表，set 儲存組成音以便快速查詢。演算法方面使用查表法從 map 對應出頻率，並用走訪和線性搜尋來處理資料。
