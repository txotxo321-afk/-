## Final Report

# 專案說明

我這次製作的是一個和弦播放器（Chord Player），使用者可以根據自己的需求，決定播放的和弦進行、樂器種類、速度（BPM）、音色演算法以及效果器參數等。

和弦可以視為多個音符同時發聲的組合，而每個音符都對應到特定的頻率。因此，聲音合成的核心目標就是根據這些頻率產生對應的波形，再將它們混合成最終的聲音。

本程式採用數位音訊的方式產生聲音，設定採樣率（Sample Rate）為 44100 Hz，也就是將一秒鐘切分成 44100 個時間點，並在每個時間點計算聲波的振幅值。算出每個時間點的振幅後，就能形成可以播放的數位音訊 sample。

## 程式流程

我的程式流程如下：

使用者輸入 → 根據輸入得出頻率和各參數資料 → 用樂器算法得出一段基礎音訊 → 疊加效果器算法得出最終音訊 → 將音訊傳進播放函式並播放

本專案將著重於樂器算法。

本專案依照樂器種類分為 Guitar、Piano 與 Rhodes 三組，每組各實作兩種不同的聲音合成演算法。這些演算法的目的都是根據輸入的頻率資料產生對應的波形，再將多個音符合成為最終聲音。不同演算法的差異在於它們模擬樂器聲音的方式不同，因此在音色特性與運算成本上也有所不同。

# 一、吉他

## 1. Karplus–Strong Synthesis

Karplus–Strong 演算法是一種經典的撥弦樂器合成方法，常被用來模擬吉他的弦振動。它的核心概念是利用延遲線（Delay Line）（琴弦過去一個週期內的振動狀態）與回授（Feedback）來模擬琴弦在被撥動後來回振動並逐漸衰減的行為。

其核心公式可表示為：

y[n] = 0.5 × (y[n-N] + y[n-N-1])

其中 N 約等於 fs / f，fs 為取樣率，f 為音符頻率。

每次從 buffer（延遲線）取出兩個相鄰 sample 做平均，再放回尾端。由於平均的過程會逐漸消除高頻成分，因此聲音會隨時間慢慢變得柔和並自然衰減，形成類似真實吉他弦的聲音。

時間複雜度為 O(SK)，其中 S 為總 sample 數量、K 為同時發聲的音符數量（和弦的音符數）。

空間複雜度為 O(KL)，其中 L 約等於 fs / f，代表每個音符所需的延遲線長度，也就是 sample rate / frequency。

產生一個 C4_M7、tone=2.0、delay=0、bpm=120、bar=1 的音訊所需時間（9 次取平均）：15.866 ms

---

## 2. Extended Karplus–Strong Synthesis

Extended Karplus–Strong 是對基本 Karplus–Strong 的改良版本，目的是讓聲音更接近真實吉他，並改善基本模型的一些限制。這個版本加入了 Fractional Delay、Damping，以及 Pick Position 等機制。

其核心可簡化表示為：

current = (1 - α)x0 + αx1

next = feedback × ((1 - d)current + d × prev)

其中：

x0 是 buffer 目前位置的 sample  
x1 是下一格 sample  
α 為插值比例，用來實作 Fractional Delay  
d 為 damping 參數，用來模擬高頻衰減  
feedback 為回授係數

Fractional Delay 的作用是讓延遲長度不必侷限在整數 buffer 的長度，因而能更準確地控制音高。由於實際頻率對應的週期長度通常不是整數，因此只使用整數長度的 buffer 會造成音高誤差，而 fractional delay 可以利用插值的方式改善這個問題。

Damping 用來模擬弦振動時能量的損失。真實琴弦在震動時，高頻成分通常會比低頻更快衰減，因此透過低通濾波可以讓高頻逐漸消失，產生更自然的吉他聲音。

Pick Position 則是模擬不同撥弦位置所造成的頻譜差異。若撥弦位置靠近琴橋，高頻泛音會較明顯，聲音會比較亮；若撥弦位置靠近琴頸，聲音則會較柔和。程式透過改變初始雜訊的分布來模擬這種效果，使音色變化更豐富。（目前 pick position 是固定的，未來可以擴充功能讓它成為可變動的參數）

時間複雜度為 O(SK)，與基本 Karplus–Strong 相同，但常數較大。

空間複雜度為 O(KL)，與基本版本類似，但每個 voice 需要額外的狀態變數。

實際產生所有 sample 的時間（9 次取平均、參數同上）：9.594 ms

### 觀察

1. 兩者聽感差不多。
2. 雖然 Extended Karplus–Strong 在理論上加入了 Fractional Delay 與 Damping Filter，使單次計算較複雜，但實際 Benchmark 結果顯示其執行速度反而優於原始 Karplus–Strong。推測是因算法 1 使用 deque 作為延遲線，而算法 2 使用 vector 與 circular buffer 實作，具有較佳的記憶體區域性，因此降低了記憶體存取成本，使整體執行時間反而較短。

# 二、鋼琴

## 1. FM Synthesis

FM（Frequency Modulation）Synthesis 是一種利用兩個振盪器合作產生聲音的方法。其中一個振盪器（carrier）負責產生最終聽到的聲音，另一個振盪器（modulator）則負責改變前者的振動方式。這樣產生出的波形會比一般正弦波複雜，包含更多泛音，因此能夠模擬鋼琴這類音色較豐富的樂器。（振盪器可以視為一個持續產生週期性波形的數學函數）

其核心公式可表示為：

y(t) = sin(ωc t + I sin(ωm t))

其中：

ωc 為 Carrier 頻率  
ωm 為 Modulator 頻率  
I 為 Modulation Index

在本專案中，FM 鋼琴使用兩個振盪器：Carrier 與 Modulator。Carrier 負責輸出主要聲音，而 Modulator 則負責調變 Carrier 的相位或頻率。當調變深度增加時，聲音中的高頻成分也會變多，音色就會更亮、更有穿透力。程式中也搭配 envelope，使聲音在起音後逐漸衰減，以模擬鋼琴按鍵被敲擊後的自然消音過程。

FM Synthesis 的優點是運算量低、參數少卻能產生複雜聲音，而且非常適合電子鋼琴或數位合成鋼琴。不過它的音色控制方式比較抽象，參數調整需要多次嘗試。

時間複雜度為 O(SK)，因為每個 sample 只需要計算固定數量（2 個）的振盪器狀態。（其中 S 為總 sample 數量、K 為同時發聲的音符數量，也就是和弦的音符數。）

空間複雜度為 O(K)，只需儲存每個音的 carrier phase 與 modulator phase。

實際產生 sample 的時間（9 次取平均）：40.234 ms

## 2. Additive Synthesis with Inharmonicity

Additive Synthesis 是透過對每個音符生成多個不同頻率的正弦波（例如基頻 + 2 倍頻 + 3 倍頻…），再把它們加總起來，就會形成比較接近真實鋼琴的音色。

其核心公式可表示為：

y(t) = Σ An sin(2π f n t)

其中 n 表示第 n 個泛音。

然而，真實鋼琴弦並不是理想的細線，因為琴弦具有剛性，因此其泛音頻率不完全是基頻的整數倍，而是會有些微偏移，這種現象稱為 Inharmonicity。為了模擬這個特性，程式在計算每個泛音時加入修正項，使泛音頻率略高於理想值：

f n = n f √(1 + B n²)

其中 B 為 Inharmonicity 係數。這樣產生的聲音會比單純的正弦波疊加更接近真實鋼琴。

這個方法的好處是能夠非常明確地控制鋼琴聲音的亮度、厚度與衰減特性，因此在模擬真實鋼琴時相當實用。缺點則是如果泛音數量很多，計算量會比 FM 更大。

時間複雜度為 O(SKH)，其中 H 為 harmonic 數量；雖然 H 在程式中是固定常數（7），但每個 sample 都要額外計算多個泛音，因此常數較大。

空間複雜度為 O(KH)，需要為每個音保存每個 harmonic 的相位。

實際產生 sample 的時間（9 次取平均）：152.995 ms

### 觀察

1. 二者聽感其實差不大。
2. FM synthesis 每個 sample 僅需計算兩個振盪器（carrier 與 modulator），運算量較低；而 Additive synthesis 需對每個音符計算多個泛音（7 個 harmonics），並包含額外的頻率修正與衰減計算，因此在相同 sample 數下具有更高的計算成本，導致執行時間顯著增加。

# 三、Rhodes（電鋼琴）

## 1. Additive Synthesis

Rhodes 電鋼琴與一般鋼琴不同，它的聲音來自金屬音叉（Tine）被槌子敲擊後的振動，再經由拾音器轉換成電訊號。因此 Rhodes 的音色通常帶有金屬感、鈴聲感與溫暖的尾音。

本專案的 Rhodes Additive Synthesis 是透過多個 partial 疊加來模擬 Rhodes 的音色。其核心公式可表示為：

y(t) = Σ An e^(-λn t) sin(2π r n f t)

其中：

An 為第 n 個 partial 的振幅  
λn 為第 n 個 partial 的衰減速度  
rn 為第 n 個 partial 的頻率比例

partial 指的是構成聲音的各個頻率成分；harmonic 是其特例（整數倍關係）。這裡因涵蓋多種非理想頻譜模型（非整數倍），因此統一使用 partial 作為描述單位。

這些 partial 的頻率不一定是整數倍，因此不完全屬於傳統鋼琴那種理想泛音列，而比較接近 Rhodes 金屬振動所產生的頻譜特性。程式中針對每個 partial 設定不同的振幅與衰減速度，使高頻成分在起音時較明顯，之後快速消失，只留下主體聲音。

時間複雜度為 O(SKP)，其中 P 為 partial 數量；這次的 P 為固定常數（P=6）。

空間複雜度為 O(KP)，需要為每個 partial 儲存對應的 phase。

實際產生 sample 所需時間：287.852 ms

## 2. Modal Synthesis

Modal Synthesis 是一種較偏向物理建模的方法。它將聲音來源視為多個振動模態的組合，並將其組合。這種方法非常適合模擬金屬物體或共鳴體，因為真實物體的振動通常不是單一頻率，而是由多個自然模態共同構成。

模態是指物體在受激振動時所呈現的不同自然振動方式，每一種模態對應一個特定的共振頻率。在金屬或複雜結構（如 Rhodes 音叉）中，聲音可視為多個模態的疊加，而非單純的整數倍泛音列。

其核心公式可表示為：

y(t) = Σ Am e^(-λm t) sin(2π fm t)

其中：

Am 為第 m 個模態的振幅  
λm 為第 m 個模態的衰減速度  
fm 為第 m 個模態的頻率

在本專案中，Rhodes 的金屬音叉（Tine）被視為由數個振動模態所組成。每個模態具有不同的頻率比例、初始振幅與衰減速率，其中高次模態衰減較快，低次模態則維持較久。這樣的設計可以模擬 Rhodes 在敲擊時先出現明亮鈴聲感。

相較於單純的 Additive Synthesis，Modal Synthesis 更接近實際樂器的物理行為，因此在某些情況下能產生更自然的音色。不過它的計算與建模方式也相對複雜。

時間複雜度為 O(SKM)，其中 M 為 mode 數量；這次的 M 為固定常數（M=6）。

空間複雜度為 O(KM)，需要保存每個模態的相位與狀態。

實際產生 sample 所需時間：195.626 ms

### 觀察

1. 兩者在最終音色有一定差異，直接比較兩者有點不太公平。
2. 雖然兩者皆為 O(S·K) 的時間複雜度，但算法 1 需額外計算 click、attack、airy 與 tremolo 等音色修飾項，使每個 sample 的常數成本較高，因此整體運算時間明顯增加。

# 四、效果器

除了上述六個演算法外，還有兩個參數可供使用者自行調整，分別是 Tone 與 Delay。這兩個效果都是在樂器模擬演算法完成後再進行疊加。

## 1. Tone（音色亮度控制）

### 功能

Tone 用來調整聲音的明亮程度（Brightness）。

較小的 Tone 值會削弱高頻成分，使聲音較柔和、較暗；較大的 Tone 值則會強調高頻成分，使聲音較明亮。

### 實現方法

當 Tone 小於 1 時，程式會使用一階平滑濾波器，讓高頻成分逐漸減弱，使音色變得較柔和。其公式可表示為：

y[n] = a x[n] + (1 - a) y[n-1]

其中：

x[n] 為目前 sample  
y[n] 為輸出 sample  
a 為 Tone 參數

當 Tone 大於 1 時，程式會計算相鄰 sample 的差值，利用此差值近似高頻訊號，再將其加入原始訊號中，以達到強化高頻成分的效果，使音色變得較明亮。其公式可表示為：

y[n] = x[n] + g · hp

其中：

g 為增益係數  
hp = x[n] - x[n-1]

而這個差值主要代表高頻波形資訊。

Tone 的時間複雜度為 O(N)，空間複雜度為 O(N)，其中 N 為音訊 sample 數量。

---

## 2. Delay（回聲效果）

### 功能

Delay 用來模擬回聲（Echo）效果。

聲音播放後，系統會在稍後的時間再次播放較小音量的相同訊號，使聲音聽起來更有空間感與延續感。Delay 值越大，回聲效果越明顯。

### 實現方法

程式會先設定一個固定的延遲時間，並將其換算成對應的 sample 數量。接著在產生新的 sample 時，除了保留原始訊號外，也會混入較早之前的訊號，並乘上一個衰減係數後加入輸出結果。

其基本概念可表示為：

y[n] = x[n] + k · x[n-D]

其中：

x[n] 為原始訊號  
x[n-D] 為 D 個 sample 之前的訊號  
k 為回聲衰減係數

由於回聲訊號的音量會逐漸衰減，因此能模擬真實空間中聲音反射後逐漸減弱的效果。

Delay 的時間複雜度為 O(N)，空間複雜度為 O(N)，其中 N 為音訊 sample 數量。

經過上述步驟後，就完成了聲音的合成。

# 五、使用方式

我是使用 Visual Studio Code 終端機進行編譯與執行。

執行後，使用者可選擇：

- Benchmark：測試各演算法的執行效能與記憶體使用情況。
- Play Loop：實際播放合成出的聲音。

在 Play Loop 模式下，使用者需要依序輸入：

1. 和弦數量（作為循環播放的一組和弦）
2. 各和弦代號（例如 C4_M7_11_5；C4 是和弦基底音，M7 是和弦種類，11 是要增加的音，5 是要減去的音，並以底線分隔）
3. 樂器種類（Guitar、Piano 或 Rhodes）
4. 速度（BPM，每分鐘拍數）
5. 播放小節數（Bars）
6. 合成演算法（Algorithm 1 或 Algorithm 2）
7. Tone 參數（控制音色明亮程度）
8. Delay 參數（控制回聲效果）

程式會根據輸入的參數進行聲音合成，並播放對應的和弦循環。
